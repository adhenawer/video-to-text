INTRODUÇÃO

Daval é co-fundador de uma consultoria de IA e passou anos trabalhando em empresas como Bloomberg e Nvidia. Ele ajudou dezenas de engenheiros de software a fazer a transição para funções de engenharia de IA, e com isso entregou mais de 25 projetos de IA para clientes nos EUA e nos Emirados Árabes.

Este roteiro é construído para engenheiros de software com pelo menos 2 anos de experiência que querem migrar para engenharia de IA — um programa de 8 semanas, com 4 a 5 horas por dia, 5 a 6 dias por semana, usando apenas recursos gratuitos.

A premissa central: engenheiros de software são os mais bem posicionados para se tornar engenheiros de IA. A fórmula é: habilidades sólidas de backend + compreensão de LLMs e modelos de IA + design de sistemas de IA.

SEMANA 1: FUNDAMENTOS DE IA E PYTHON

Python é a linguagem da IA — não há debate sobre isso. Mas se você já programa em Java, .NET ou qualquer outra linguagem, não precisa aprender Python do zero. Use transferência de aprendizado: mapeie os conceitos que você já conhece (loops, OOP, controle de fluxo, exceções) para a sintaxe Python. Não gaste seis meses aprendendo Python e FastAPI do zero.

A ferramenta essencial para aprender sem gastar dinheiro é o Groq — uma plataforma que permite chamar vários modelos de LLM gratuitamente via API. Você cria uma API key gratuita, integra ao Python, e aprende a fazer chamadas a LLMs sem pagar nada durante a fase de aprendizado.

O que aprender na prática: como chamar APIs de LLM pelo Python, FastAPI para construir backends (análogo ao Spring Boot ou ASP.NET), conceitos básicos de IA generativa (o que é um agente, parâmetros de configuração de LLM como temperatura e max tokens), embeddings e bancos de dados vetoriais (começar com Chroma open source, mas saber que na indústria usam Qdrant, Pinecone).

Soft skills desta semana: criar perfis profissionais no LinkedIn e no Twitter/X. Presença online é o novo currículo — não basta ter uma conta que você atualiza quando está procurando emprego.

Tarefa prática: construir um servidor FastAPI simples, um motor de busca básico com Chroma ou Qdrant.

SEMANA 2: FUNDAMENTOS DE RAG E LANGCHAIN

60 a 70% dos projetos de IA em produção são projetos de RAG (Retrieval Augmented Generation) — você aponta o LLM para o seu repositório interno de dados e ele trabalha sobre esse conhecimento. Essa é a base da maioria das aplicações empresariais de IA.

O que aprender: pipeline completo de RAG, parsing e chunking de documentos com Docling, fundamentos do LangChain para interagir com modelos de chat, como conectar a knowledge base ao LLM.

Uma ressalva importante sobre frameworks: LangChain faz muitas atualizações. É muito útil entender como construir um pipeline de RAG em Python puro primeiro — sem o framework — para que você consiga depurar quando as abstrações falharem.

Soft skills desta semana: começar a seguir personalidades proeminentes em IA (como Andrej Karpathy) e engajar de forma significativa nos posts deles. Comentários bem elaborados com milhares de pessoas de IA vendo constroem relacionamentos que eventualmente abrem portas — para emprego, para negócios, para colaboração. Presença online é uma forma nova de currículo.

O conceito de “produto engenheiro” está emergindo: uma fusão do engenheiro de software com o product owner. 50% habilidades técnicas, 50% conhecimento de produto e negócios. Domain knowledge específico (finanças, saúde) é cada vez mais valioso.

SEMANA 3: FUNDAMENTOS DE IA AGÊNTICA

Nesta semana você aprende a construir agentes de IA reais. O ciclo fundamental é o ReAct (Reasoning + Action): o agente pensa, decide que ação tomar, executa a ação com uma ferramenta, observa o resultado, e repete.

Roteamento em agentes: um chatbot de suporte ao cliente precisa detectar a intenção do usuário e rotear para o especialista correto. Também precisa rotear com base na complexidade — perguntas simples vão para modelos baratos, perguntas que exigem raciocínio profundo vão para o modelo mais poderoso.

Segurança de agentes: na prática, a primeira pergunta que todo cliente faz é “meus dados estão seguros?” Você precisa saber as respostas — como configurar guardrails, como processar dados localmente quando necessário, como evitar vazamento de informações sensíveis.

Avaliação de sistemas de IA: sistemas de IA são probabilísticos, não determinísticos. Fazer o mesmo teste cinco vezes vai dar respostas diferentes. Você precisa de um paradigma de avaliação diferente do testing tradicional de software — frameworks específicos para medir a qualidade de saídas de LLMs ao longo do tempo.

Soft skills desta semana: aprender a fazer apresentações que não adormecem o público. A habilidade de comunicar tecnicamente para stakeholders não técnicos é crítica — especialmente quando o cliente chega com uma arquitetura técnica montada no ChatGPT e confiança de que está certo.

SEMANA 4: OBSERVABILIDADE E DEPLOY

LangSmith (da família LangChain) é a ferramenta de observabilidade: você rastreia execuções, vê traces, monitora custos por chamada, analisa threads de conversação. Indispensável para entender o que está acontecendo em produção.

Deploy de sistemas agênticos é diferente do deploy de software tradicional. Por causa da natureza probabilística dos LLMs, o paradigma de deployment muda. AWS Agent Core é uma plataforma específica para deploy de agentes na AWS.

Contribuição open source: ao contratar engenheiros de IA, a primeira coisa que chama a atenção num currículo é “5 PRs merged em repositórios open source”. Menos de 1% dos currículos que Daval recebe têm isso. Contribuir ficou muito mais fácil com IA — você encontra issues marcadas como “good first issue” no GitHub, usa Claude Code ou ChatGPT para entender o problema e propor solução.

SEMANA 5: SISTEMAS MULTI-AGENTE E ENGENHARIA DE CONTEXTO

LangGraph é o framework para quando você quer controle profundo sobre sistemas agênticos — em oposição ao LangChain que abstrai muita coisa. A analogia: LangChain é como gravar vídeo em modo automático, LangGraph é o modo manual de uma DSLR. Mais controle, mais complexidade, resultados profissionais.

Sistemas multi-agente envolvem múltiplos agentes especializados trabalhando em conjunto — cada um com seu domínio de responsabilidade, orquestrados por um agente coordenador. CrewAI é outra opção de framework para isso.

Engenharia de contexto é a disciplina de gerenciar eficientemente o que você coloca no contexto do LLM — saber o que incluir, o que omitir, como comprimir informação sem perder relevância, como otimizar o uso de tokens.

Tarefa prática: construir um sistema multi-agente para automatizar tarefas de marketing, construir um servidor MCP para uma API pública.

SEMANA 6: ENGENHARIA DE IA AVANÇADA

Otimização de custo: contas de API de LLM podem explodir em produção sem que você perceba — análogo às surpresas de fatura na AWS. As técnicas principais são: caching de respostas, rate limiting, seleção de modelo por complexidade da tarefa (modelo barato para tarefas simples, modelo caro apenas quando necessário), e model cascading.

RAG multimodal: em setores como saúde e manufatura, há muito uso de visão computacional. RAG multimodal vai além de texto — inclui imagens, vídeos, áudio. É uma especialização valiosa.

Fine-tuning: quando RAG sozinho não é suficiente, você pode precisar ajustar o próprio modelo com LoRA, QLoRA e outras técnicas de fine-tuning eficiente em parâmetros.

Modelos locais: muitos clientes (especialmente no Oriente Médio e em setores regulados) exigem que os dados não saiam da infraestrutura deles. Ollama é o framework principal para rodar modelos localmente — seja num laptop, seja em servidores com GPUs dedicadas.

SEMANA 7: IA DECLARATIVA COM DSPY

DSPy é um framework que permite construir aplicações de IA usando programação declarativa — você declara o que quer que o sistema faça, e o framework otimiza automaticamente os prompts e parâmetros para atingir o objetivo. Para engenheiros de software com background em programação declarativa, o conceito vai ser familiar.

SEMANA 8: DEPLOY EM NUVEM

As duas nuvens mais relevantes para projetos de IA corporativa são Azure e AWS. Google Cloud (com Vertex AI e Google AI Studio) também é importante, mas aprender uma nuvem bem torna as outras mais fáceis — os conceitos são similares, só mudam os nomes.

Fundamentos necessários: resource groups, subscriptions, regiões, storage, IAM (Identity and Access Management). A lógica de transferência de aprendizado se aplica aqui também — se você já conhece AWS, migrar para Azure é muito mais rápido do que aprender do zero.

DICAS PARA APRENDIZADO EFETIVO

Forme um grupo de estudos com colegas engenheiros que também querem fazer a transição. Check-ins semanais, responsabilidade mútua, compartilhamento de aprendizados — aprender em grupo é significativamente mais fácil do que aprender sozinho.

A regra de ouro do aprendizado eficiente: menos tempo consumindo informação, mais tempo digerindo, implementando e compartilhando. Assistir vídeos passivamente não basta — você precisa construir coisas, se deparar com problemas reais, e explicar o que aprendeu para outros.

Use Claude Code ou ChatGPT como tutor pessoal durante toda a jornada de aprendizado. Quando não entender algo, pergunte. Quando travar num erro, descreva o contexto e peça ajuda. Esses modelos são tutores infinitamente pacientes com conhecimento profundo.

O mais importante: as habilidades técnicas estão ficando menos diferenciadas (qualquer agente pode escrever código), enquanto as soft skills — comunicação, gestão de stakeholders, apresentação, personal branding, domínio de negócio — estão ficando cada vez mais valiosas. A combinação de 30% de habilidades técnicas sólidas com 70% de soft skills bem desenvolvidas cria profissionais verdadeiramente difíceis de substituir.
