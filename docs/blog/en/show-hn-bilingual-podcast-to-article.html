Show HN: A bilingual podcast-to-article pipeline built on Claude Code

video-to-text · 2026-04-13

I watch a lot of AI/dev podcasts and I can't stand re-scrubbing a 90-minute video to find the five paragraphs I cared about. So I built a pipeline that turns YouTube and Twitter/X videos into readable articles in both Portuguese and English, publishes them as plain static HTML, and exposes a Markdown endpoint for agents. 21 articles are live today at adhenawer.net (PT) and adhenawer.net/en (EN).

What it does

Give it a URL. It detects the provider, pulls the transcript (YouTube captions when available, Whisper otherwise), hands the raw text to Claude to translate and reorganize into thematic sections, and writes a semantic HTML file with JSON-LD, OG tags, hreflang, and a sitemap entry. No CMS, no database, no build step — just a Python script that drops a file into docs/leituras/ and GitHub Pages serves it.

The whole flow is driven from the Claude Code CLI. I paste a URL into the agent, it runs the pipeline, writes the translation, builds the HTML, updates the index and sitemap, commits, pushes. I review the diff. That's the loop.

Tech stack

  • Transcription: mlx-whisper (whisper-large-v3-turbo) on Apple Silicon; faster-whisper on Windows/Linux. Audio is pulled via yt-dlp.
  • YouTube captions: youtube-transcript-api when the video has them — skips Whisper entirely.
  • Translation + sectioning: Claude (via the Claude Code agent). A local fallback exists using mlx-vlm with Gemma, but Claude is noticeably better at preserving technical nuance and naming sections sensibly.
  • Output: hand-written HTML template in Python (no Jinja), ~400 lines. Emits <article>, <section>, JSON-LD Article, hreflang alternates, and a sepia-by-default reader theme (sepia exists because pure white triggers halation in astigmatic eyes — roughly half the population).
  • Hosting: GitHub Pages + a Cloudflare Worker on the free tier.
  • Tests: pytest, TDD on the providers, HTML generator, and slide extractor.

The thing I'd call out on HN

There's a Cloudflare Worker sitting in front of the site that strips HTML to clean Markdown on the fly. curl https://adhenawer.net/en/article.md returns about 75% fewer tokens than the HTML equivalent. It's entirely on the free tier (100k requests/day) and the code is maybe 60 lines. The point is that if an LLM agent wants to read one of these articles, it shouldn't have to pay for <div class="…"> soup. There's also an llms.txt at the root in the proposed convention, linking every article with its sections and a one-line summary.

Combined with the bilingual structure (same slug in PT and EN, hreflang alternates in both directions), the result is: human readers get a comfortable sepia page in their language, and crawlers/agents get a machine-friendly Markdown twin of the same content at a predictable URL.

Try it

  • Live demo (PT): https://adhenawer.net
  • Live demo (EN): https://adhenawer.net/en/
  • Repo: github.com/adhenawer/video-to-text
  • Markdown for agents: append .md to any article URL.

To add an article locally: python3 src/pipeline.py URL --title '…' --slug '…'. The README covers the Twitter/X path and the optional slide-extraction mode (ffmpeg + OpenCV pulls distinct slides from screen-recorded talks and interleaves them with the text by timestamp).

Known limits

  • Best path is Apple Silicon. On a 16GB M1 Pro, mlx-whisper transcribes a 90-minute talk in a few minutes. The Windows/Linux path works but I use it less, so rough edges are likely.
  • Whisper struggled noticeably on one Mandarin video about order-flow trading — worse than on English/Portuguese sources. I kept the article but the transcript needed hand-fixes.
  • Translation quality depends on Claude being in the loop. The local Gemma fallback works, but loses nuance on technical jargon often enough that I only use it offline.
  • The "agent adds an article" flow is nice but not idempotent by itself — I still eyeball the diff before pushing. Mostly to catch section titles Claude invented out of enthusiasm.
  • No search yet. 21 articles is fine with Ctrl-F on the index; at 100 it won't be.

What's next

Client-side search over the llms.txt index (lunr or a tiny prebuilt inverted index). A proper RSS feed in both languages. More providers — Spotify and Apple Podcasts episodes that expose audio would be a natural fit. And I want to try having the agent suggest a slug, title, and TL;DR from the transcript itself instead of me typing them in.

Happy to answer questions about the Whisper setup, the Worker, or the Claude Code loop. Honest feedback welcome — especially on the reading experience.
