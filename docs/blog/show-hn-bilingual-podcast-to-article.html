Show HN: Um pipeline bilíngue de podcast para artigo construído sobre o Claude Code

video-to-text · 2026-04-13

Eu assisto muito podcast de IA/dev e não suporto ficar rebobinando um vídeo de 90 minutos para achar os cinco parágrafos que me interessaram. Então construí um pipeline que transforma vídeos do YouTube e Twitter/X em artigos legíveis em português e inglês, publica como HTML estático puro, e expõe um endpoint Markdown para agentes. 21 artigos já estão no ar em adhenawer.net (PT) e adhenawer.net/en (EN).

O que ele faz

Você dá uma URL. Ele detecta o provider, pega a transcrição (legendas do YouTube quando disponíveis, Whisper caso contrário), entrega o texto bruto para o Claude traduzir e reorganizar em seções temáticas, e escreve um arquivo HTML semântico com JSON-LD, tags OG, hreflang e uma entrada no sitemap. Sem CMS, sem banco de dados, sem etapa de build — só um script Python que solta um arquivo em docs/leituras/ e o GitHub Pages serve.

O fluxo inteiro é conduzido pelo CLI do Claude Code. Eu colo uma URL no agente, ele roda o pipeline, escreve a tradução, gera o HTML, atualiza o index e o sitemap, faz commit e push. Eu reviso o diff. É esse o loop.

Stack técnica

  • Transcrição: mlx-whisper (whisper-large-v3-turbo) em Apple Silicon; faster-whisper em Windows/Linux. O áudio vem via yt-dlp.
  • Legendas do YouTube: youtube-transcript-api quando o vídeo tem — pula o Whisper inteiro.
  • Tradução + seccionamento: Claude (via agente do Claude Code). Existe um fallback local usando mlx-vlm com Gemma, mas o Claude é visivelmente melhor em preservar nuance técnica e nomear seções de forma sensata.
  • Saída: template HTML escrito à mão em Python (sem Jinja), ~400 linhas. Emite <article>, <section>, JSON-LD Article, alternates de hreflang, e um tema de leitura sépia por padrão (sépia existe porque branco puro dispara halation em olhos com astigmatismo — algo como metade da população).
  • Hospedagem: GitHub Pages + um Worker do Cloudflare no plano gratuito.
  • Testes: pytest, TDD nos providers, no gerador de HTML e no extrator de slides.

A parte que eu destacaria no HN

Tem um Worker do Cloudflare na frente do site que remove o HTML e entrega Markdown limpo na hora. curl https://adhenawer.net/en/article.md retorna cerca de 75% menos tokens que o HTML equivalente. Roda inteiramente no plano gratuito (100k requisições/dia) e o código tem uns 60 linhas. A ideia é que, se um agente LLM quiser ler um desses artigos, ele não deveria pagar por sopa de <div class="…">. Também tem um llms.txt na raiz seguindo a convenção proposta, linkando cada artigo com suas seções e um resumo de uma linha.

Combinado com a estrutura bilíngue (mesmo slug em PT e EN, alternates de hreflang nas duas direções), o resultado é: leitores humanos ganham uma página sépia confortável no idioma deles, e crawlers/agentes ganham um gêmeo Markdown amigável para máquinas do mesmo conteúdo em uma URL previsível.

Teste

  • Demo (PT): https://adhenawer.net
  • Demo (EN): https://adhenawer.net/en/
  • Repo: github.com/adhenawer/video-to-text
  • Markdown para agentes: adicione .md ao fim de qualquer URL de artigo.

Para adicionar um artigo localmente: python3 src/pipeline.py URL --title '…' --slug '…'. O README cobre o caminho do Twitter/X e o modo opcional de extração de slides (ffmpeg + OpenCV extraem slides distintos de talks gravadas de tela e intercalam com o texto por timestamp).

Limitações conhecidas

  • O melhor caminho é Apple Silicon. Num M1 Pro de 16GB, o mlx-whisper transcreve uma talk de 90 minutos em poucos minutos. O caminho Windows/Linux funciona, mas uso menos, então provavelmente tem arestas.
  • O Whisper teve dificuldade visível em um vídeo em mandarim sobre order-flow trading — pior do que em fontes em inglês/português. Mantive o artigo mas a transcrição precisou de ajustes manuais.
  • A qualidade da tradução depende do Claude estar no loop. O fallback local com Gemma funciona, mas perde nuance em jargão técnico com frequência suficiente para eu só usar offline.
  • O fluxo de "agente adiciona um artigo" é legal, mas não é idempotente por si só — eu ainda bato o olho no diff antes de dar push. Principalmente para pegar títulos de seção que o Claude inventou por entusiasmo.
  • Ainda sem busca. Com 21 artigos dá pra usar Ctrl-F no index; com 100 não vai dar.

Próximos passos

Busca client-side sobre o índice do llms.txt (lunr ou um índice invertido pré-construído minúsculo). Um feed RSS decente nos dois idiomas. Mais providers — episódios do Spotify e Apple Podcasts que expõem áudio seriam um encaixe natural. E quero tentar fazer o agente sugerir slug, título e TL;DR a partir da própria transcrição em vez de eu digitar.

Fico feliz em responder perguntas sobre o setup do Whisper, o Worker, ou o loop do Claude Code. Feedback honesto é bem-vindo — especialmente sobre a experiência de leitura.
