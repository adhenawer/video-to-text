[16:32] THE BIRTH OF THINKING

In 2024 OpenAI released O1, the omni model. Its trick: thinking. An LLM generates text by calculating tension over your context to produce the next token, then repeats. Before thinking, they just spat out answers, and the answers were dumb.

With thinking, the model is trained to generate more questions for itself before answering. It self-feeds. That "Thinking..." animation is it generating questions and feeding them back to itself, layer after layer. Deep thinking or high thinking controls how many layers. That is why Opus is slower than Sonnet: it thinks deeper. And that is why it burns more tokens.

[30:53] PROMPT, SYSTEM PROMPT AND TOOL CALLING

An LLM is a big file loaded into GPU memory, used to compute the next token. To drive the model you send a prompt. Several kinds. One you never see is the system prompt. The harness opens the session with a system prompt preloaded: "you are a coding assistant, you have access to tools A, B, C, D; when the user types X, respond Y". That runs before you type anything.

Then comes your prompt. It flies to the LLM over an API. The LLM does deep thinking and returns a plan. You approve. It starts executing. That is where tool calling enters: a training that became standard from 2024 to 2025. When the model needs to create a file, it returns a structure with the function `createFile("index.html")`. The harness receives that structure and runs it on your machine. The LLM runs nothing. The harness does.

================================================================================
[32:55] HOW THE HARNESS EXECUTES

If the LLM wants to read a directory, it sends `listDir("/")`. The harness runs it, captures the output, and feeds it back to the LLM as a new prompt. Before 2024 we did this by hand: copy the error, paste in the chat, get a response, copy the code, paste in the terminal. Tool calling automated everything. The harness is dumb: receives a function, executes, returns. The intelligence lives in the model; the execution lives in the harness. That is why a good harness matters.

================================================================================
[35:41] EVERY HARNESS SPEAKS ITS OWN LANGUAGE

Each company defines its own proprietary tool calling format. On OpenAI it is `create_file`. On Anthropic, `file_new`. On GLM, `new_file`. It is like Java, Python, and JavaScript. A harness has to speak all of them. Open Code, which is open source, tries to speak them all or to ask the model to reply in a format it understands. If the model was not trained on that format, it has to translate, and performance degrades.

That is why Cloud Code goes with Opus, Codex with GPT, Gemini with Gemini CLI or Antigravity. Technically you can mix them. You would just be being dumb.

[50:00] DEEPSEEK IS GARBAGE

DeepSeek came out early 2025 and everyone cried revolution. Today nobody talks about it. To play in this generation you need four integrated things: deep thinking; KV cache (store token compute to avoid recomputing — every model has had this forever); prompt caching (cache the history of the whole session); tool calling trained inside deep thinking; and mixture of experts (partition parameters so only the active slice loads into memory).

DeepSeek does not have prompt caching: it is slower because it recomputes everything. Shallow deep thinking. Outdated tool calling, 2024-era. The outcome: 40 tokens per second, while OpenAI and Anthropic run at 400 to 500. Anything below 100 tokens per second is unusable. Doesn't matter how good the answer is — you just cannot use it.

[54:58] QUANTIZATION AND SWAP: NO MAGIC

[1:45:50] VARIANTS AND DISTILLATION: "FLAVOR"

[1:54:41] THE DEFINITION OF AGENT AND SKILL

An agent is a program that uses an LLM and runs the functions it requests. Cloud Code spins up multiple agents in parallel.
