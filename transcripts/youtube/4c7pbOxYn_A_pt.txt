[16:32] O NASCIMENTO DO THINKING

Em 2024 a OpenAI lançou o O1, o modelo omni. A grande sacada dele foi o thinking. Uma LLM gera texto calculando a tensão em cima do seu contexto para gerar o próximo token, e repetindo a conta. Antes do thinking, ele gerava resposta direto, e as respostas eram burras.

Com thinking, ele é treinado para, antes de responder, gerar novas perguntas para si mesmo. Fica se retroalimentando. Aquele momento em que o chat mostra "Thinking..." é ele gerando perguntas e devolvendo para ele mesmo, camada após camada de resposta. Deep thinking ou high thinking é quantas camadas profundas ele pode ir. Por isso Opus é mais lento que Sonnet: pensa mais fundo. E por isso gasta mais token.

[30:53] PROMPT, SYSTEM PROMPT E TOOL CALLING

LLM é um arquivão que sobe na memória da GPU para calcular o próximo token. Para manipular o modelo, você manda um prompt. Tem vários tipos. O que você não vê se chama system prompt. A rédea abre a sessão já com um system prompt: "você é um assistente de programação, tem acesso às ferramentas A, B, C, D; toda vez que o usuário fala X, você responde Y". Já tem série de instruções pré-carregadas.

Daí você manda seu prompt. Vai via API para o LLM, que faz deep thinking. Ele devolve um plano. Você aprova. Ele começa a executar. Nesse ponto entra o tool calling: treinamento que começou a ficar padrão em 2024-2025. Quando o modelo precisa criar um arquivo, ele devolve uma estrutura com a função `createFile("index.html")`. O harness recebe essa estrutura e executa na sua máquina. O LLM não executa nada. Quem executa é o harness.

================================================================================
[32:55] COMO O HARNESS EXECUTA

Se o LLM quer ler um diretório, ele manda `listDir("/")`. O harness executa, captura a saída e devolve como novo prompt para o LLM. Antes de 2024, a gente fazia isso manualmente: copiava o erro, colava no chat, esperava resposta, copiava o código, colava no terminal. Tool calling automatizou tudo. O harness é burro: só recebe função, executa, devolve. A inteligência toda está no modelo, mas a execução fica no harness. Por isso harness precisa ser bom.

[50:00] DEEPSEEK É UMA BOSTA

DeepSeek saiu no começo de 2025 e todo mundo achou que era revolução. Hoje ninguém fala mais dele. Para estar nesta geração você precisa de quatro coisas integradas: deep thinking; KV cache (guardar o cálculo para não recalcular a cada token — todos os modelos têm isso desde o começo); prompt caching (guardar o cache de múltiplas requisições do histórico); tool calling treinado para chamar ferramentas dentro do deep thinking; e mixture of experts (particionar os parâmetros para carregar só a parte ativa na memória).

[2:25:12] PROMPT LEAKING E PROMPT INJECTION

Você cola corpo de e-mail num prompt para "avalie esses e-mails". Se num dos e-mails tiver algo que parece prompt, modelo mal treinado obedece. Antigamente, em aplicações web, foi SQL injection: campo de review do e-commerce vira insert into comentários. Se o texto do review vem bruto, o cara põe aspas e `DELETE FROM tabela`. Apaga tudo. Hoje, prompt injection é análogo.

Sanitização: tudo que vem do usuário é perigoso. Toda aspa vira barra aspa. Todo `<` vira `&lt;`. É HTML entity. Se pedir ao LLM "pegue o comentário e grave", ele vai gravar igual qualquer um. Precisa instruir ele a sanitizar. E se for só para você na sua máquina, não precisa — contexto importa. Eu não vou fazer meu software de casa igual Netflix faz site dela.

[2:58:42] AI JAIL: MÁQUINA VIRTUAL VS CONTAINER

Máquina virtual vira uma máquina dentro da outra. Peso de dois sistemas operacionais. Não é necessário. O que a gente usa é container. Todo programa na sua máquina enxerga todos os outros programas: o Word sabe que tem o Firefox rodando, e tem acesso aos diretórios.

Container não é máquina virtual: roda na mesma CPU, no mesmo kernel. Se tem bug no kernel, o programa sai da jail. Uma máquina virtual tem hypervisor e outro nível, o que é mais seguro, mas mais pesado. Entendendo máquina virtual e container você sabe escolher.
